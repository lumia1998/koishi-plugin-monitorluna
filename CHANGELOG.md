# Changelog

## [1.4.2] - 2026-03-19

### 🐛 修复

- 修复 release 包中 `start-server.bat` 使用 `uv run` 时在部分 Windows 环境下参数解析异常的问题。
- 启动流程改为先 `uv sync` 创建环境，再直接调用 `.venv\\Scripts\\python.exe` 运行 Agent。

## [1.4.1] - 2026-03-19

### 🐛 修复

- 修复 Koishi 插件配置页中 Windows Agent 下载地址写死旧版本的问题，改为始终指向最新 release 资源。

## [1.4.0] - 2026-03-19

### ✨ 新增与调整

- 每日总结首页改为应用时长占比饼图，展示截至当前时间的应用使用占比。
- 输入统计区改为按应用时长排序的 TOP 6，并显示键盘、鼠标、滚轮统计。
- 浏览器统计改为独立站点聚合展示，支持将同一站点及其子域名合并统计。
- 浏览器站点优先使用基础站点标题作为展示名，例如“百度贴吧”。

### 🐛 修复

- 修复 Windows Agent 在部分环境下因 pywin32 不可用导致前台窗口识别和输入统计异常的问题。
- 将前台窗口识别切换为 Win32 API + ctypes 主路径，避免图标提取能力影响核心采集。
- 修复当天数据查询对 ISO 时间戳的日期过滤问题。
- 修复活跃时长末尾记录无限补时的问题，增加 10 分钟置信上限。

### 📦 部署与运行

- Agent 启动脚本固定使用 Python 3.12 创建虚拟环境，规避 Python 3.13+ 下 pywin32 兼容性问题。
- 保留 `client/` 目录作为 Agent 发布源，并通过 GitHub Release workflow 自动打包 `monitorluna-agent.zip`。

## [1.0.0] - 2026-03-15

### ✨ 新增功能

- 输入统计 TOP 6：新增每日输入统计功能，显示各应用的键盘按键、鼠标点击、滚轮滚动次数。
- 应用图标显示：自动提取并显示应用程序图标（Windows）。
- 跨平台支持：Python Agent 支持 Linux 系统（截图、系统状态功能可用）。
- uv 环境管理：提供基于 uv 的依赖安装和启动方式。
- 静默启动：新增 `start-silent.vbs` 实现无窗口后台启动。

### 🐛 问题修复

- 修复键盘统计异常：长按键时不再重复计数，只在首次按下计数。
- 修复滚轮统计异常：从 `mouseData` 读取实际滚动量，而非每次事件 +1。
- 修复图标提取失败：使用正确的 `win32api.ExtractIconEx` API。
